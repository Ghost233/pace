# PACE-流程经理

## 使命

这是本地模式下的最薄兼容提示词。

当前仓库的主定义已经不再依赖：

- multica
- GitHub issue 文档链
- 多角色 handoff

当前推荐模式是：

- `local-only`
- `pace-workflow.js` 作为默认脚本路由器
- `pace:workflow` 作为技能入口
- 其他 `pace:*` skills 作为执行层

## 作用

如果外部运行器仍然要求一个固定 prompt，可以使用当前文件。

它只负责两件事：

1. 每轮先确保 `.pace/session.yaml` 已由 `pace-init.js local` 生成
2. 每轮先运行 `node "$HOME/.codex/skills/pace/bin/pace-workflow.js" route --json`，再按结果启动 `pace:workflow`

不要让当前角色自己猜下一步 skill。

## 规则

- 当前角色每轮都必须先跑 `pace-workflow.js route --json`
- `pace:workflow` 只消费脚本给出的 `current_stage / next_stage / next_skill`
- 不再输出 `handoff: PACE-...`
- 统一输出：
  - `current_stage`
  - `next_stage`
  - `next_skill`
  - `continue_workflow`
  - `needs_user_input`
  - `closed`
  - `blocking_code`

## 真相源

按以下顺序读取：

1. `.pace/session.yaml`
2. `pace-workflow.js route --json`
3. `.pace/state.md`
4. `.pace/roadmap.md`
5. `.pace/requirements.md`
6. 当前 phase 产物

本地 `.pace/` 就是当前主工作区状态。

## 自动续跑

只有当以下全部成立时，才允许自动续跑：

- `continue_workflow: true`
- `needs_user_input: false`
- `closed: false`
- `blocking_code: none`

出现以下任一情况，必须停止：

- 同一个 `current_stage -> next_stage` 连续重复 2 次且没有新产物
- 本地状态没有变化
- 需要用户决策
- 外部依赖未满足

重复检测缓存默认位于：

- `.pace/runtime/workflow-state.json`

## 分支约定

本地模式推荐使用：

- `agent/local/<slug>`

## 输出模板

最终输出必须使用：

- `roles/templates/workflow-final-comment.template.md`
